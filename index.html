配置Nginx作为反向代理

什么是反向代理？先说正向代理，A知道C并想和C通讯，但是A和C通讯必须通过B，B就是正向代理或者直接就叫代理。但是，如果A不知道C存在，只知道找到B就能得到回应。对于A来说，B后面有什么不重要，只需要得到C的回应。B可以根据自己的配置来决定怎么回应。正向（forward）和反向（reverse）代理并非反义，只是翻译成中文后在字面上产生了歧义。Nginx在这里就是为服务器上的web应用提供反向代理服务。

其实root和alias的区别很简单但是很重要，alias后面指定static文件的全路径，root后面指定的是static文件所在项目的根路径，他会自动在路径后面加上location指定的值。所以如果你的static等网站文件不在默认的目录下，最好还是用alias配置。

Koa2框架下在特定页面或者路由中使用socket.io的解决方案

Posted on 2018-06-21

在使用Koa2框架过程中，发现如果想在特定页面下实时输出信息，需要引用socket.io模块。
该模块本身的功能很多，除了可以向用户实时打印信息外，最重要是可以搭建类似即时网页聊天室。这个模块的原理是当用户连接上网站之后，就建立起了一个socket连接，随时发起会话。对于我的需求来说，其实只是想实现其中一个非常简单的特性，就是在其中一个页面中显示服务器上的运行信息或者输出。最开始的考虑是建立一个namespace。这样可以为指定的路径或者路由页面分配socket，起到降低TCP连接和隔离的效果。在这个namespace下面，可以添加join和leave的功能。

对于在Koa2下创建namespace的方法，用.of实现起来很简单，

const server = require('http').createServer(app.callback());
const io = require('socket.io')(server);
io
    .of('/running')
    .on('connection', function(socket){
        console.log('client connected'); //终端显示连接状态
        setInterval(function(){socket.emit('date', {'date': new Date()}, 1000);}) //页面打印当前时间
    });

同时记得在客户端页面指定

var socket = io('/running');

socket.io模块对于express框架的支持文档和内容很多，官网上的许多例子基本都是用express，关于Koa2的就比较少。

最简单的实现方式就是将io对象替换为全局（global）对象，即将

const io = require('socket.io')(server);

替换为

global.io = require('socket.io')(server);

这样，在路由页面下，直接打开io即可

io.on('connection', function(socket){
    console.log('someone connected');
    setInterval(function(){socket.emit('date', {'date': new Date()}, 1000);})
})

koa和multer构造网站文件上传功能

Posted on 2018-06-01

在使用koa和multer构造网站文件上传功能的过程中遇到了几个棘手的问题。

根据官方文档的说明，中间件中可以用以下方法调用multer：

const multer  = require('multer')
const upload = multer({ dest: 'uploads/' })
const router = require('koa-router')()

router.post('/upload', upload.single('files'), async (ctx, next)=>{})

注：ctx是有koa传入的封装了request和response的变量，可以通过ctx.req和ctx.res访问request和response，但是不支持直接调用底层res进行响应，比如res.send, res.writeHead, res.write都应该被避免。

当我在重构网站控制器的时候，上传文件功能被放在一个单独的文件upload.js而非app.js里面。在upload.js里面，应该如何使用multer实现上传功能，并利用koa-router跳转到下一个页面去？

常规的控制器本身是按照如下的方式写:

module.exports = {
    'POST /uploads': async(ctx, next) => {}
}

通过另外一个mapping函数来自动处理控制器GET，POST等动作。但是multer这里有两个中间件，upload.single和async(ctx, next)，就没有办法直接放进去。

Stack Overflow上面找到一个问答，和我的情况类似。有人给出来了两个方案：
方案一

module.exports = [upload.single('files'), async (ctx, next) {
    // Put the rest of your code here
}];

方案二

var filesUpload = upload.single('files');

module.exports = function(ctx) {
    imageUpload(ctx, function(err) {
        // Put the rest of your code here
    });
};

第二种方案是把剩下的功能（路由）作为multer中间件的callback，看起来没有第一种方案直观，所以我没有尝试。直接使用第一种方案，把两个中间件作为数组的形式expose出来。

合并成数组后运行测试报错如下：

Error: post `/uploads`: `middleware` must be a function, not `object`

再回头去网上找到了解决办法，说是koa-router不支持以数组作为中间件，多个中间件必须一个一个地输入。如果合并成数组，就会扔出来以上错误，需要调用koa-compose来解决这个问题，形式为以下：

app.get('/foo', compose([middleware1, middleware2]));

按照以上提示，文件上传成功。
